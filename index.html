HTML

  • HTML - Hypertext Markup Language
  • The head element contains information about the webpage.
  • The body element represents the visible content shown to the user.
  • meta defines metadata such as the character set, description, keywords, author and viewport
  • src attribute defines the location of an image
  • alt attribute contains a text string that descrives the apperance and functionality of the image
  • a tag is anchor and creates links to a webpage
  • href attribute points to the URL for a link
  • Takeaway: Use semantic HTML elements to indicate the role of their content, as well as to give structure to the content of the webpage

CSS

  • CSS = Cascading Style Sheets
  • A margin indicates how much space we want around the outside of an element
  • A padding indicates how much space we want around the content inside an element
  • Inline CSS - to style an element directly in the HTML file using the "style" attribute
  • Internal style sheet - contains CSS rules for the webage in the head section of the HTML file
  • External style sheet - most common, keeps all CSS rules in a seperate file, which makes design changes easier
  • Declaration - contains two important components: the CSS property we want to apply and the value of the property
  • Class attribute - allows us to share a CSS rule to any element we chosse by assiging the rule to a class attribute with a class selector: card

Git

  • git status: checks what branch we are currently on
  • git checkout -b "branch name": creates a new branch and switches to it
  • git add -A: adds all changes
  • git commit -m: simple, descriptive message on what changes were made
  • git pull origin main: to pull latest version of main branch
  • git push origin feature/"branch name": pushes feature branch from local to remote
